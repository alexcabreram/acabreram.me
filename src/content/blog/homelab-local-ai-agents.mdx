## How it all started

I bought a NAS to store files. That's it. That was the plan.

Then I thought "wouldn't it be cool if it also ran my media server?" Then came monitoring. Then a reverse proxy. Then AI agents. And here we are — a full infrastructure project that I definitely didn't plan but I'm honestly quite proud of.

If you've ever fallen down the homelab rabbit hole, you know exactly what I mean. If you haven't... well, consider this your warning. 😄

## Two machines, zero cloud bills

The whole setup runs on two boxes sitting on my desk:

A **UGREEN DXP2800 NAS** that handles storage, media automation (the whole Sonarr/Radarr stack), monitoring with Prometheus and Grafana, and reverse proxy. It's basically a tiny server pretending to be a NAS. It does a lot more than storing files at this point.

A **Mac Mini** that acts as the brain. It runs **Ollama** natively on Apple Silicon — GPU-accelerated LLM inference without paying a single API call. On top of that, **OpenClaw** orchestrates AI agents and **Open WebUI** gives me a ChatGPT-like interface to talk to local models. All of this running on a box the size of a sandwich.


## The fun part — AI agents that actually do things

This is where things get interesting. I have a private Telegram group with topic channels, and each one is connected to a different AI agent through OpenClaw:

| Agent | Channel | What it does |
|-------|---------|-------------|
| `main` | #assistant | My personal AI — answers questions, brainstorms ideas, thinks out loud with me |
| `monitor` | #monitoring | Gets Prometheus alerts via n8n — just logs and notifications, no AI chatter |
| `media` | #media | "Add The Bear season 4" → done, it talks to Jellyseerr |
| `devops` | #devops | Container health checks, Docker troubleshooting |

Everything runs on local Llama models. No data leaves my network. And when I need heavier reasoning for something complex, I switch to Azure OpenAI with a quick `/model azure/gpt-4o` and come back when I'm done. Best of both worlds.

## Monitoring everything (because I sleep better that way)

Prometheus scrapes every container and endpoint. Grafana makes it pretty. Alert rules catch disk pressure, container crashes, and memory issues before I even notice.

Uptime Kuma adds HTTP health checks — if Jellyfin goes down at 3am, my phone buzzes. Not that I'll fix it at 3am, but at least I *know*. There's comfort in that.

## The media stack (a.k.a. why my family puts up with the NAS)

Sonarr, Radarr, Prowlarr, Bazarr, Jellyfin, Jellyseerr. Everyone at home can request movies and series from their phones without knowing any of this infrastructure exists.

When it works — and it usually does — nobody notices. When something breaks, everyone notices immediately. That's invisible infrastructure for you.

## Things that bit me along the way

Every homelab has its "I just spent 4 hours debugging this" stories. Here are mine — hopefully they save someone else some time:

**UGOS Pro blocks port 53 on 0.0.0.0.** I couldn't figure out why AdGuard Home wouldn't start. The OS reserves that port. You need to bind to the specific interface IP. Took me way longer than I'd like to admit.

**OrbStack ≠ Docker Desktop.** On the Mac Mini, `host.docker.internal` doesn't resolve reliably. The fix is embarrassingly simple — just use the actual LAN IP in your env vars. But it's not obvious when you're deep in Docker networking at midnight.

**Docker images with hardcoded UIDs on UGOS Pro.** The SSH user can't `chown` Docker volumes because the daemon runs as root but you don't. Workaround? Run `chown` inside a throwaway Alpine container. Not pretty, but it works.

**Local LLMs are better than I expected.** For infrastructure tasks — summarizing logs, answering Docker questions, drafting alerts — a 7B model on Apple Silicon is faster than any API roundtrip. I genuinely expected to depend on Azure OpenAI much more than I actually do.
